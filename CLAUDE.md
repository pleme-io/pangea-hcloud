# pangea-hcloud

Hetzner Cloud provider resources for the Pangea infrastructure DSL. All resource
code is **auto-generated** by `pangea-forge` via `iac-forge-cli`. Manual edits
to resource files will be overwritten on the next generation run.

## Structure

```
lib/
  pangea-hcloud.rb                    # Entry point (generated -- requires all resources)
  pangea-hcloud/version.rb            # Version constant (manual)
  pangea/
    resources/
      types/hetzner/
        core.rb                       # Provider-specific Dry::Types (generated)
      hcloud.rb                       # Aggregator module (generated)
      hcloud_*/                       # 29 resource directories (generated)
        types.rb                      # Dry::Struct attributes class
        resource.rb                   # ResourceBuilder DSL
spec/
  spec_helper.rb                      # Test setup (manual)
  support/                            # Shared test helpers
  resources/                          # RSpec stubs (generated)
```

## Generated vs Manual

| File | Source |
|------|--------|
| `lib/pangea/resources/hcloud_*/` | Generated by pangea-forge |
| `lib/pangea-hcloud.rb` | Generated (require file) |
| `lib/pangea/resources/hcloud.rb` | Generated (aggregator) |
| `lib/pangea/resources/types/hetzner/` | Generated (type definitions) |
| `spec/resources/` | Generated (RSpec stubs) |
| `lib/pangea-hcloud/version.rb` | Manual |
| `spec/spec_helper.rb` | Manual |
| `Gemfile`, `*.gemspec`, `flake.nix` | Manual |

## Resource count

- **29 resources** (servers, networks, subnets, routes, firewalls, load balancers,
  floating IPs, primary IPs, volumes, SSH keys, snapshots, placement groups,
  certificates, DNS zones, DNS zone record sets, rDNS, etc.)

## Dependencies

- pangea-core ~> 0.2

- dry-types ~> 1.7
- dry-struct ~> 1.6

## Testing

```sh
bundle exec rspec                            # Run all tests
bundle exec rspec spec/resources/            # All resource specs
```

## Regeneration

Resources are generated by `pangea-forge` (the Pangea backend of `iac-forge`).
Run regeneration via `iac-forge-cli`:

```sh
terraform-schema-importer generate \
  --schema schema.json \
  --provider "registry.terraform.io/hetznercloud/hcloud" \
  --provider-name hcloud \
  --output-dir .
```
